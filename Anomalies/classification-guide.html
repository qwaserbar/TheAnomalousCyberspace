Anomaly Classification Guide


Almost every anomaly documented within The Anomalous Cyberspace has a classififcation table thats purpose is to give the said anomaly an appropriate overview of what it is, and what is being dealt with.

There are at least 3 things that the classification table gives an insight into:

  • Manifestation; What form is the anomaly in? [File, Website, Database, Other]
  • Threat towards the world; How dangerous of an impact can the anomaly have on our world? [On a scale of 0 to 5]
  • Status; How is the anomaly currently? [Active, Dormant, Extinct, Unknown]

Manifestation


Mainfestation describes the form of the Anomaly. This ranges from a file on a computer, a publicly accessible website, a database full of something, or something else entirely.


If the anomaly is a File, then list the file format, and byte count. and if the anomaly is a compressed archive, [Examples include .zip, .rar, .tar, etc], then list the byte count of the file when it is both compressed and uncompressed.

An example of this can be seen in Anomaly 001 - "BaseMaker.zip"'s classification.




If the anomaly is a Website, then list the domain extension



If the anomaly is a Database, then there is nothing else to list out.



If the anomaly is something else entirely, then listing other details may vary.



Threat Towards the world


Threat Towards the world describes how the anomaly can dangerously impact the world.

This is written as a scale of 0 to 5



Status


The status of an anomaly tells how the anomaly is doing currently.


The Status of an Anomaly can be:
  • Active: The Anomaly is still appearing commonly to this day
  • Dormant: The Anomaly is inactive, but however it can become active again sometime in the future.
  • Extinct: The Anomaly is inactive and will stay inactive forever.
  • Unknown: It is unknown if the Anomaly is Active, Dormant, or Extinct.
